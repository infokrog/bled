## 🏠 The Public Demonstrator: 400-Student Hostel Scheduling

To provide the scientific and open-source community with an immediate, verifiable proof-of-concept, we have included a single-file web demonstration: `hostel_matrix.html`.

### Problem Statement
* **Input Data:** 400 unique students (200 Boys, 200 Girls).
* **Constraints:** Students must be organized into 100 rooms of four (two roommate pairs per room). Boys and girls can never share a room, but their single-gender rooms can dynamically interleave in any order throughout the hostel sequence (`Ro-1` to `Ro-100`).
* **The Complexity:** The space of all valid, non-mixing room allocations is staggeringly large (extending well over $10^{550}$ options).

### How the $\Phi$-Map Engine Solves It On-The-Fly:
1. **Dynamic Room Interleaving:** The engine utilizes a precise combinatorial bisection to determine the gender layout sequence of the 100 rooms.
2. **Two-Stage Polynomial Matching:** For each gender space, the algorithm computes the total exact space of unique pairs ($n \times (n-1)/2 = 19,900$ basic pairs per gender) and maps them into a two-phase perfect matching network (Phase A: Students $\to$ Pairs; Phase B: Pairs $\to$ Rooms of 4) in polynomial time.
3. **The Counter Matrix ($C_m$) Display:** On the user interface, you can watch the mechanical-proportional gear shifts ($m_1$ to $m_5$) translate any short integer query instantly into full, verified room layouts.

## 🛠️ Key Technical Features in the Artifact

* **Zero Dependencies:** Written in pure, clean HTML5 and Vanilla JavaScript. No `npm`, `pip`, or Flask servers required. Double-click to run natively in any browser.
* **Infinite-Precision Navigation:** Built leveraging JavaScript `BigInt` data types, ensuring the 560+ digit addresses are translated with absolute bit-level accuracy without rounding errors.
* **On-the-Fly Verification & Performance Tracking:** The script features an independent verification loop that cross-checks the output against fundamental constraints (gender separation, unique student allocation, exact room capacities). It tracks and displays both **Query Generation Time** and **Checking Time** in fractions of a millisecond ($P$-time).
* **Dynamic CSV Logger:** Features an automated logging mechanism that formats solutions into a clean CSV format, preserving column priorities (`Room Number`, `Gender`, `Pairs Data`, `Execution Time`, `Global Index`) with structured spacing for immediate analytical processing.
